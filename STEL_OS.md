# STEL OS — North Star Document

> *From goal to published campaign in minutes. AI-driven. Brand-consistent. Human-approved.*

---

## What Is STEL OS?

STEL OS is an AI-powered marketing operating system that automates the full lifecycle of a marketing campaign — from strategy through content creation, visual production, scheduling, publishing, and performance analysis.

It is not a content generator. It is not a scheduler. It is not a CRM. It is all of them, orchestrated by AI agents that work together as a coordinated team — one that understands your brand, your audience, and your goals.

When a marketer or founder opens STEL OS, they describe what they want to achieve in plain language. The platform takes it from there: building strategy, writing platform-native copy, generating on-brand visuals, scheduling posts at optimal times, publishing to connected channels, and reporting back on what worked.

---

## The Problem

Modern marketing requires managing a sprawling, fragmented stack. Strategy in one tool. Copy in another. Design in another. Scheduling in another. Analytics in another. CRM somewhere else. Each handoff between tools — and between people — introduces delays, inconsistencies, and dropped context.

Even with a full team, the process is slow:

1. A campaign idea takes days to become a strategy
2. A strategy takes more days to become written content
3. Written content needs design, then approval, then scheduling
4. Publishing is manual or requires a separate platform
5. Analytics are siloed and rarely feed back into the next campaign

For solo founders, small marketing teams, and agencies managing multiple clients, this cycle is unsustainable. The good work — the creative thinking, the relationship-building, the strategic bets — gets crowded out by execution overhead.

**STEL OS collapses this entire workflow into a single, unified system.** The AI handles execution. The human stays in control of decisions.

## The Six-Stage Pipeline

At the core of STEL OS is an autonomous, six-stage AI pipeline. It runs in the background, produces real deliverables, and surfaces them for human review before anything goes live.

```
GOAL DEFINED
    │
    ▼
┌─────────────────────────────────────────────────────┐
│  Stage 1 — STRATEGY                                  │
│  Marketing Strategist Agent analyzes goal, audience, │
│  timeline, and budget. Produces: executive summary,  │
│  target personas, key messages, KPI targets,         │
│  30-day action plan, channel budget allocation.      │
└─────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────┐
│  Stage 2 — CAMPAIGN                                  │
│  Campaign record created and linked to strategy.     │
│  Channels confirmed. A/B variants assigned.          │
└─────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────┐
│  Stage 3 — CONTENT                                   │
│  Content Creator Agent writes platform-native copy   │
│  for each selected channel: LinkedIn, Twitter/X,     │
│  Instagram, Facebook, TikTok, Email, Blog, SMS,      │
│  Google Business. Each piece respects that           │
│  platform's character limits, tone, and format.      │
│  Brand voice, banned phrases, and UTM tags applied.  │
└─────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────┐
│  Stage 4 — IMAGES                                    │
│  Image Generator Agent creates visuals for each      │
│  channel using Fal.ai Flux. Prompts are informed by  │
│  the content, brand brief, and channel dimensions.   │
└─────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────┐
│  Stage 5 — COMPOSITING                               │
│  Compositor overlays brand elements on each image:   │
│  logo, primary/secondary colors, headline text, CTA. │
│  Output is a channel-sized, production-ready asset.  │
└─────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────┐
│  Stage 6 — COMPLETE                                  │
│  All assets ready. Scheduled posts pre-built with    │
│  ML-optimized send times per channel. Human reviews, │
│  approves, and launches.                             │
└─────────────────────────────────────────────────────┘
    │
    ▼
LIVE CAMPAIGN → ANALYTICS → OPTIMIZATION → NEXT CAMPAIGN
```

Every stage is idempotent — if interrupted, the pipeline resumes without duplication. Stages are persisted as they complete, so no work is lost.

## Feature Set

### Goals & Campaigns

- **Quick Campaign**: Type a single sentence. The system infers goal type, recommends channels, and queues the pipeline.
- **Goal Types**: Leads, Awareness, Conversions, Traffic, Social growth, Product launch, Event

### Brand Kit

Everything in STEL OS is brand-aware. The Brand Kit is the single source of truth for how a company looks and sounds.

- Logo upload and position control (auto, top-left, top-right, bottom-left, bottom-right)
- Primary and secondary brand colors
- Font preference (modern, serif, minimal, bold)
- Brand voice profile: tone, values, messaging guidelines
- Banned hashtags and content moderation rules
- AI-extracted brand brief: mood, style, color palette derived from uploaded inspiration

Every piece of content, every image, every composited asset uses this kit. The result is campaign-to-campaign consistency without a designer in the loop.

## Architecture Overview

STEL OS is built as a monorepo with four primary layers:

```
apps/
  web/       — Next.js 14 frontend (dashboard, API routes, compositor)
  api/       — Express backend (all business logic, integrations, auth)

packages/
  agents/    — All AI agent definitions (Claude-powered)
  queue/     — Inngest job orchestration (pipeline, CRM jobs, analytics)
  db/        — Drizzle ORM schema and migrations (PostgreSQL)
  compositor/ — Image compositing (brand overlays, channel sizing)
  integrations/ — Platform clients (LinkedIn, Twitter, Meta, Twilio, etc.)
```

**Key architectural decisions:**
- **Inngest** for background job execution — event-driven, resume-able, observable
- **Drizzle ORM** for type-safe database access with full migration history
- **Idempotent pipeline stages** — every step checks DB state before running; safe to retry
- **Multi-tenant** — all queries scoped to `orgId`; complete org isolation
- **Encrypted token storage** — OAuth tokens encrypted at rest
- **Role-based access** — owner, admin, editor, viewer with per-endpoint enforcement

**AI/ML services:**
- Claude (Anthropic) — all content, strategy, analysis agents
- Fal.ai Flux Schnell — image generation
- Satori + resvg — server-side SVG/PNG compositor for brand overlays

## What Makes STEL OS Different

**1. It is a complete loop, not a point tool.**
Most AI marketing tools stop at content generation. STEL OS takes that content all the way to publishing and brings the performance data back in to inform the next campaign. The loop is closed.

**2. It knows your brand.**
The Brand Kit is not cosmetic. It informs the content agents (voice, banned phrases, messaging themes), the image generator (brand brief, color palette, mood), and the compositor (logo, colors, position). Every output is on-brand without manual intervention.

**3. Platform-native content, not copy-paste AI slop.**
The Content Creator Agent has channel-specific instructions for each platform. A LinkedIn post sounds like LinkedIn. A TikTok script reads like TikTok. Instagram captions have hashtags in the right place. The character limits, tone requirements, and structural conventions of each platform are baked in.

**4. Human control at every gate.**
The pipeline is autonomous, but nothing publishes without human approval unless the user has explicitly enabled auto-publish. Every asset passes through review. Every scheduled post can be edited or cancelled.

**5. AI learns from your results.**
The Optimization Agent analyzes what actually performed and feeds those learnings back into the system — best posting times, top-performing messages, budget reallocation. Each campaign makes the next one better.

**6. Deterministic where it should be.**
Pre-flight checks, character limits, brand safety scans — these run without AI. Fast, cheap, reliable. AI is used where judgment is needed, not where rules suffice.
